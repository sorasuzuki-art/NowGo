# NowGo 詳細設計書

## 1. プラン生成アルゴリズム詳細

### 1.1 全体フロー

```
searchSpotsFromDB(params)
  │
  ├─ 1. 訪問済みスポット取得（visited_spots）
  ├─ 2. DB検索（バウンディングボックス + フィルタ）
  ├─ 3. スコアリング（scoreSpot）
  ├─ 4. 多様性選択（selectDiverseSpots）
  ├─ 5. ルート最適化（optimizeRoute）
  └─ 6. タイムテーブル生成（buildPlan）
```

### 1.2 DB検索（ステップ2）

```sql
SELECT * FROM spot
WHERE "isActive" = true
  AND indoor_type IN ({locationType条件})
  AND lat >= {origin.lat - latDelta}
  AND lat <= {origin.lat + latDelta}
  AND lon >= {origin.lng - lonDelta}
  AND lon <= {origin.lng + lonDelta}
LIMIT 500
```

**バウンディングボックス計算**:
```
walkKm = walkRangeMinutes × 80 / 1000   （例: 20分 → 1.6km）
latDelta = walkKm / 111
lonDelta = walkKm / (111 × cos(origin.lat × π / 180))
```

**フォールバック**: 候補0件の場合、バウンディングボックスなしでLIMIT 300で再検索

### 1.3 スコアリング（ステップ3）

#### 基本除外条件（return null）
- name, lat, lon いずれかが欠損
- 営業時間外
- excludeSpotIds に含まれる
- visitedIds に含まれる

#### スコア計算

基本スコア = 1.0 に対して以下を乗算:

**天気補正**:
| 天気 | indoor_type | 乗数 |
|------|------------|------|
| 雨/雪 | outdoor | ×0.1 |
| 雨/雪 | indoor | ×1.4 |
| 雨/雪 | weather_ok含む'雨OK' | ×1.3 |
| 晴れ | outdoor/both | ×1.2 |

**滞在時間妥当性**:
```
estimatedSpots = max(2, round(availableTime / 35))
perSpotTime = availableTime / estimatedSpots
if (stayMin > perSpotTime × 2)   → ×0.5
if (stayMin > perSpotTime × 1.5) → ×0.8
```

**スタイル補正**:
| スタイル | stay_type | 乗数 |
|----------|----------|------|
| ゆっくり | stay | ×1.3 |
| ゆっくり | short | ×0.8 |
| アクティブ | roam | ×1.3 |
| アクティブ | short | ×1.2 |
| アクティブ | outdoor | ×1.1 |
| ほどほど | short | ×1.1 |

**モード補正（最重要）**:

定番:
```
score = 0.8 + famousLevel × 0.15
```

新規開拓:
```
if famousLevel >= 5   → return null（除外）
if scope === 'big'    → return null（除外）
if category === 'landmark' or 'town' → return null（除外）
if famousLevel <= 2   → ×1.4
if famousLevel === 4  → ×0.4
if scope === 'medium' → ×0.5
```

冒険:
```
if famousLevel >= 5   → return null（除外）
if scope === 'big'    → return null（除外）
if category === 'landmark' or 'town' → return null（除外）
if group in ['いきもの','エンタメ','リラックス','ミュージアム'] → ×1.3
if scope === 'medium' → ×0.5
```

**タグボーナス**:
| 条件 | タグ | 乗数 |
|------|------|------|
| style=ゆっくり | のんびり, リラックス, 静か | ×1.15 |
| mode=新規開拓 | 穴場, 隠れ家, ローカル | ×1.15 |

**価格帯補正**: free→×1.2, low→×1.1, medium→×1.0, high→×0.9

**カテゴリ別時間帯補正**:
| カテゴリ | 時間帯 | 乗数 |
|----------|--------|------|
| bar | 17時前 | ×0.05（ほぼ除外） |
| bar | 20時以降 | ×1.4 |
| bar | 17-19時 | ×1.0 |
| cafe | 22時以降 | ×0.5 |
| restaurant | ランチ(11-14時) / ディナー(17-21時) | ×1.1 |

### 1.4 多様性選択（ステップ4）

#### 目標スポット数
```
targetSpots = clamp(round(availableTime / 35), 2, 6)
newSpotsNeeded = max(1, targetSpots - pinnedSpots.length)
poolCount = min(newSpotsNeeded + 2, 8)
```

#### 1つ目の選択
```javascript
// 全候補から完全均等ランダム
const first = candidates[Math.floor(Math.random() * candidates.length)]
```

#### 2つ目以降の選択

各候補のcomboスコアを計算し、重み付きランダムで選択:

```
combo = √(score) × proximityScore × diversityBonus

proximityScore = max(0.1, exp(-distance_km × 0.7))
diversityBonus  = 既出カテゴリ ? 0.05 : 1.0
```

**距離減衰テーブル**:
| 距離(km) | proximityScore |
|----------|---------------|
| 0.5 | 0.70 |
| 1.0 | 0.50 |
| 1.5 | 0.35 |
| 2.0 | 0.25 |
| 3.0 | 0.12 |

**重み付きランダム選択**:
```javascript
function weightedRandomPickFromPool(pool, weights) {
  total = sum(weights)
  r = random() × total
  for each (item, weight) in (pool, weights):
    r -= weight
    if r <= 0: return item
  return last item
}
```

### 1.5 ルート最適化（ステップ5）

#### ビル名パース
```
住所: "東京都渋谷区神宮前4-12-10 表参道ヒルズ3F"
           ↓ 正規表現でビル名抽出
ビル名: "表参道ヒルズ"
```
パターン: `番地の後のスペース以降` からフロア情報（B?数字F/階）を除去

#### グルーピング
- 同一ビル名のスポットを1ノードにまとめる
- 1ノード = 1つの代表座標（最初のスポットの座標を使用）

#### 全順列探索（TSP + 方向転換ペナルティ）
```
ノード数N（通常2〜6）に対して N! 通りの順列を全探索

for each permutation:
  totalDist = distance(origin → perm[0])
  for i = 1 to N-1:
    totalDist += distance(perm[i-1] → perm[i])

  // 方向転換ペナルティ（行ったり来たり防止）
  for i = 1 to N-2:
    angle1 = atan2(perm[i-1] → perm[i])
    angle2 = atan2(perm[i] → perm[i+1])
    angleDiff = |angle2 - angle1|
    if angleDiff > 90度:
      totalDist *= 1 + (angleDiff / 180度) × 0.3

  if totalDist < bestDist:
    bestDist = totalDist
    bestOrder = permutation
```

#### 同カテゴリ連続解消（ルート最適化後）
```
ルート確定後、隣接スポットが同カテゴリの場合:
  後方の異カテゴリスポットとスワップ
  → カフェ→カフェ→バー を カフェ→バー→カフェ に並び替え
```

### 1.6 タイムテーブル生成（ステップ6）

```
startMinute = ceil(minute / 15) × 15   // 15分単位に切り上げ
deadline = startTime + availableTime × 1.1  // 10%余裕

for each spot in routed:
  walkMin = 同一ビル ? 0 : max(3, ceil(距離m / 80))
  stayMin = spot.estimated_stay_min ?? 30  // DB値をそのまま使用

  if !isPinned:
    endTime = currentTime + walkMin + stayMin
    if spots.length >= 2 && endTime > deadline → break

  spot.time = format(currentTime + walkMin)
  spot.duration = stayMin
  currentTime += walkMin + stayMin
```

## 2.5 スポットキャッシュ

DB検索結果をメモリにキャッシュし、再検索・再作成時のレスポンスを高速化する。

```
キャッシュキー = 丸め座標(小数3桁) + walkRangeMinutes + locationType
  例: "35.681,139.767,20,both"

TTL = 10分

searchSpotsFromDB():
  1. キャッシュキーを生成
  2. キャッシュヒット → DB検索をスキップ
  3. キャッシュミス → Supabase検索 → キャッシュ保存
  4. buildPlan() はキャッシュ有無に関わらず毎回実行（ランダム性を保つ）
```

座標を小数3桁で丸めることで、約100m以内の再検索はキャッシュから返す。
スコアリング・選択・ルート最適化はキャッシュ対象外（毎回実行で多様性を維持）。

## 6. 距離・時間計算

### 6.1 2点間距離（Haversine公式）

```
R = 6371 km（地球の半径）
dLat = (lat2 - lat1) × π / 180
dLon = (lon2 - lon1) × π / 180
a = sin²(dLat/2) + cos(lat1×π/180) × cos(lat2×π/180) × sin²(dLon/2)
距離 = R × 2 × atan2(√a, √(1-a))
```

### 6.2 徒歩時間

```
速度 = 80m/分 (4.8km/h)
walkMin = ceil(距離m / 80)
最低3分
```

## 7. 営業時間判定

```
isOpen(spot, hour, minute):
  if is_open_24h → true
  if starttime or closetime が未設定 → true（不明 = 営業中扱い）

  cur = hour × 60 + minute
  open = starttime の分換算
  close = closetime の分換算

  if close < open:   // 日付跨ぎ（例: 18:00-02:00）
    return cur >= open || cur < close
  else:
    return cur >= open && cur < close
```

## 9. プラン実行フロー詳細

### 9.1 スポット間移動

```
handleNext():
  1. currentSpotIndex < spots.length - 1 の場合のみ
  2. currentSpotIndex++ （DB操作なし）
  3. scrollTo top

handlePrev():
  1. currentSpotIndex > 0 の場合のみ
  2. currentSpotIndex-- （DB操作なし）
  3. scrollTo top
```

### 9.2 exitMode による完了分岐

```typescript
exitMode: 'complete' | 'abort'

// 「プランを完了する」ボタン → exitMode = 'complete'
// 「終了する」ボタン / ヘッダー戻る → exitMode = 'abort'

handleComplete():
  if exitMode === 'complete':
    // 全スポットを訪問済みに
    markSpotsVisited(plan.spots.map(s => s.id))
    // プラン履歴を保存
    savePlanHistory(plan)
  else: // abort
    // 現在のスポットまでを訪問済みに
    markSpotsVisited(plan.spots.slice(0, currentSpotIndex + 1).map(s => s.id))
    // プラン履歴は保存しない
  setScreen('dashboard')
```

### 9.3 ボトムバーの状態

| currentSpotIndex | 左ボタン | 中央 | 右ボタン |
|------------------|---------|------|---------|
| 0 | なし | 「終了する」テキスト | 次のスポットへ |
| 1 〜 N-2 | ＜（前へ） | - | 次のスポットへ |
| N-1（最後） | ＜（前へ） | - | プランを完了する |
