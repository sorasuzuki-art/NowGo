## 1. 概要

### 1.1 本ドキュメントの目的

本ドキュメントは、NowGo Web Applicationにおけるプラン提案機能（`lib/spotSearch.ts`）の全アルゴリズム・条件分岐・数式・閾値を網羅的に記述した詳細設計書である。実装コードと1対1で対応する精度を目標とし、開発・保守・レビュー時の参照資料として使用する。

### 1.2 機能概要

ユーザーが入力した「遊べる時間」「天気」「スタイル」「モード」等の条件に基づき、近隣スポットをスコアリング・選択・ルート最適化し、タイムテーブル付きのプランとして返却する機能である。

### 1.3 処理の全体像

```
ユーザー入力 → DB検索(キャッシュ付) → スコアリング → 多様性選択 → ルート最適化 → タイムテーブル生成 → プラン返却
```

### 1.4 関数一覧

|--------|------|------|

## 3. 処理フロー全体図

### 3.1 メインシーケンス

```
searchSpotsFromDB(params)
│
├─[1] 訪問済みスポット取得
│   └─ userId が存在する場合: visited_spots テーブルから spot_source_id 一覧を取得
│   └─ userId が未指定 or エラー時: visitedIds = []
│
├─[2] キャッシュ確認
│   ├─ buildCacheKey() でキー生成
│   ├─ getCachedSpots() でキャッシュヒット判定
│   ├─ ヒット時: キャッシュのDbSpot[]を使用 → [5]へスキップ
│   └─ ミス時: [3]へ
│
├─[3] DB検索（Supabase）
│   ├─ isActive = true フィルタ
│   ├─ locationType による indoor_type フィルタ
│   ├─ origin 指定時: バウンディングボックス範囲フィルタ
│   ├─ LIMIT 500
│   └─ 0件フォールバック: isActive = true, LIMIT 300（位置フィルタなし）
│
├─[4] キャッシュ保存
│   └─ spotCache.set(key, { data, timestamp })
│
└─[5] buildPlan(spots, params, visitedIds)
    │
    ├─[5.1] 除外セット構築
    │   └─ excludeSpotIds + visitedIds の和集合
    │
    ├─[5.2] ピン留めスポット変換
    │   └─ PlanSpot → ScoredSpot（score = 10 固定）
    │
    ├─[5.3] スコアリング（新規候補のみ）
    │   ├─ excludeSet に含まれるスポットをスキップ
    │   ├─ pinnedIds に含まれるスポットをスキップ
    │   ├─ isOpen() で営業時間外をスキップ
    │   └─ scoreSpot() で各スポットをスコアリング（null返却時は除外）
    │
    ├─[5.4] 目標スポット数算出
    │   └─ targetSpots = clamp(round(availableTime / 35), 2, 6)
    │
    ├─[5.5] 多様性選択（selectDiverseSpots）
    │   ├─ newSpotsNeeded = max(1, targetSpots - pinned.length)
    │   ├─ poolCount = min(newSpotsNeeded + 2, 8)
    │   └─ selectDiverseSpots(candidates, poolCount, origin)
    │
    ├─[5.6] ルート最適化（optimizeRoute）
    │   └─ optimizeRoute([...pinnedScored, ...newSelected], origin)
    │
    └─[5.7] タイムテーブル生成
        ├─ startMinute = ceil(minute / 15) * 15
        ├─ deadline = startTime + availableTime * 1.1 * 60 * 1000
        ├─ 各スポットの移動時間・滞在時間を積算
        ├─ ピン留めスポットは必ず含める
        ├─ 新規スポットは deadline 超過時に打ち切り（最低2件は保証）
        └─ PlanSpot[] を返却
```

### 3.2 内部関数呼び出し関係図

```
searchSpotsFromDB
  ├── buildCacheKey
  ├── getCachedSpots
  └── buildPlan
        ├── isOpen
        ├── scoreSpot
        │     ├── getCategoryGroup
        │     └── parseBuildingName
        ├── selectDiverseSpots
        │     ├── distanceKm

```

## 4. DB検索条件

### 4.1 メインクエリ

| 条件 | SQL相当 | 説明 |
|------|---------|------|
| 有効スポットのみ | `WHERE isActive = true` | 無効化されたスポットを除外 |
| 屋内外フィルタ | 下表参照 | `locationType` パラメータに基づく |
| バウンディングボックス | 下表参照 | `origin` 指定時のみ適用 |
| 件数上限 | `LIMIT 500` | パフォーマンス制限 |

### 4.2 屋内外フィルタ（indoor_type条件）

| locationType | indoor_type フィルタ | 説明 |
|---|---|---|
| `'屋内'` | `IN ('indoor', 'both')` | 屋内スポットと屋内外両対応スポット |
| `'屋外'` | `IN ('outdoor', 'both')` | 屋外スポットと屋内外両対応スポット |
| `undefined`（未指定） | フィルタなし | 全てのindoor_typeを返却 |

### 4.3 バウンディングボックス計算

origin が指定されている場合、以下の計算によりバウンディングボックスを設定する。

```
入力:
  walkRangeMinutes（デフォルト: 20分）
  origin.lat, origin.lng

計算:
  walkKm = walkRangeMinutes × 80 / 1000  [km]

  latDelta = walkKm / 111  [度]
  lonDelta = walkKm / (111 × cos(origin.lat × π / 180))  [度]

フィルタ条件:
  lat >= origin.lat - latDelta  AND
  lat <= origin.lat + latDelta  AND
  lon >= origin.lng - lonDelta  AND
  lon <= origin.lng + lonDelta
```

**具体例（walkRangeMinutes = 20, origin = 東京駅 35.6812, 139.7671）:**

```
walkKm = 20 × 80 / 1000 = 1.6 km

latDelta = 1.6 / 111 ≈ 0.01441 度
lonDelta = 1.6 / (111 × cos(35.6812 × π / 180))
        = 1.6 / (111 × 0.8124)
        ≈ 0.01774 度

lat範囲: [35.6668, 35.6956]
lon範囲: [139.7494, 139.7849]
```

### 4.4 フォールバック検索

| 条件 | 動作 |
|------|------|
| メインクエリ結果が `null` または 0件 | `isActive = true` のみでバウンディングボックスなし、`LIMIT 300` で再検索 |
| フォールバックも 0件 | 空配列 `[]` を返却 |

### 4.5 訪問済みスポット取得

| 条件 | 動作 |
|------|------|
| `userId` が指定されている | `visited_spots` テーブルから `user_id` で `spot_source_id` を取得 |
| `userId` が未指定 | `visitedIds = []`（除外なし） |
| クエリでエラー発生 | `visitedIds = []`（エラー無視） |

## 5. キャッシュ機構

### 5.1 キャッシュ仕様

| 項目 | 値 |
|------|-----|
| データ構造 | `Map<string, SpotCacheEntry>` |
| 保存内容 | `{ data: DbSpot[], timestamp: number }` |
| TTL | **10分**（600,000ミリ秒） |
| 無効化方式 | 遅延無効化（`getCachedSpots` 呼出時にTTLチェック） |
| スコープ | プロセス内メモリ（インスタンス変数） |

### 5.2 キャッシュキー生成規則

```
キー形式: "{latKey},{lngKey},{walkRangeMinutes},{locationType}"

latKey = origin ? (Math.round(origin.lat × 1000) / 1000).toFixed(3) : "none"
lngKey = origin ? (Math.round(origin.lng × 1000) / 1000).toFixed(3) : "none"
walkRangeMinutes = params.walkRangeMinutes ?? 20
locationType = params.locationType ?? "both"
```

**キー生成例:**

| origin | walkRangeMinutes | locationType | キー |
|--------|-----------------|--------------|------|
| `{lat: 35.6812, lng: 139.7671}` | 20 | `'屋内'` | `"35.681,139.767,20,屋内"` |
| `{lat: 35.6812, lng: 139.7671}` | 15 | `undefined` | `"35.681,139.767,15,both"` |
| `undefined` | 20 | `'屋外'` | `"none,none,20,屋外"` |

**座標丸めの精度:**
- 小数第3位で四捨五入 → 約100m単位の同一視
- 100m以内の異なる座標は同一キャッシュにヒットする

### 5.3 キャッシュの適用範囲

| 処理 | キャッシュ適用 | 理由 |
|------|--------------|------|
| DB検索（バウンディングボックス内のスポット取得） | **適用** | 同一エリアの再検索を高速化 |
| スコアリング | **毎回実行** | 天気・時間帯等で結果が変化する |
| 多様性選択 | **毎回実行** | ランダム性を保つ |
| ルート最適化 | **毎回実行** | 選択スポットが毎回異なる |
| タイムテーブル生成 | **毎回実行** | 選択スポットが毎回異なる |

## 6. スコアリング詳細

### 6.1 概要

`scoreSpot` 関数は、1つのDbSpotを受け取り、パラメータに基づいてスコアを算出する。スコアは基本スコア `1.0` に対して各種補正係数を乗算していく方式である。

```
最終スコア = 1.0 × 天気補正 × 滞在時間妥当性補正 × スタイル補正 × モード補正 × タグボーナス × 価格帯補正 × 時間帯補正
```

戻り値: `ScoredSpot | null`（null は除外を意味する）

### 6.2 除外条件（return null）

以下のいずれかに該当する場合、そのスポットは即座に除外される（null返却）。

| # | 条件 | 判定コード | 除外理由 |
|---|------|-----------|---------|
| E1 | `name` が falsy | `!spot.name` | スポット名なしは表示不可 |
| E2 | `lat` が falsy | `!spot.lat` | 座標なしはルート計算不可 |
| E3 | `lon` が falsy | `!spot.lon` | 座標なしはルート計算不可 |
| E4 | `lat` が数値変換不可 | `isNaN(parseFloat(spot.lat))` | 不正データ |
| E5 | `lon` が数値変換不可 | `isNaN(parseFloat(spot.lon))` | 不正データ |
| E6 | モード「新規開拓」で `famousLevel >= 5` | `params.mode === '新規開拓' && fame >= 5` | 超有名スポットは新規開拓の趣旨に合わない |
| E7 | モード「新規開拓」で `scope === 'big'` | `params.mode === '新規開拓' && scope === 'big'` | 商業ビル・エリアは除外 |
| E8 | モード「新規開拓」で `category === 'landmark'` or `'town'` | 同上の条件 | ランドマーク・まち歩きは除外 |
| E9 | モード「冒険」で `famousLevel >= 5` | `params.mode === '冒険' && fame >= 5` | 同上 |
| E10 | モード「冒険」で `scope === 'big'` | `params.mode === '冒険' && scope === 'big'` | 同上 |
| E11 | モード「冒険」で `category === 'landmark'` or `'town'` | 同上の条件 | 同上 |

**注意:** 営業時間外のスポット除外は `scoreSpot` 内ではなく、呼び出し元の `buildPlan` で `isOpen()` 判定済みである。`scoreSpot` に渡されるスポットは既に営業時間内であることが保証されている。

### 6.3 デフォルト値

| フィールド | デフォルト値 | 適用タイミング |
|---|---|---|
| `estimated_stay_min` | `30`（分） | `spot.estimated_stay_min ?? 30` |
| `famousLevel` | `1` | `spot.famousLevel ?? 1` |

### 6.4 天気補正

```
基本スコアに対する乗算係数
```

#### 決定表

| 天気 | indoor_type | weather_ok | 補正係数 | 説明 |
|------|-------------|------------|---------|------|
| `'雨'` or `'雪'` | `'outdoor'` | - | **×0.1** | 屋外は雨天で大幅減点 |
| `'雨'` or `'雪'` | `'indoor'` | - | **×1.4** | 屋内は雨天でブースト |
| `'雨'` or `'雪'` | その他(`'both'`等) | `'雨OK'` を含む | **×1.3** | 雨対応の屋内外両対応 |
| `'雨'` or `'雪'` | その他(`'both'`等) | `'雨OK'` を含まない | ×1.0 | 補正なし |
| `'晴れ'` | `'outdoor'` or `'both'` | - | **×1.2** | 晴天は屋外ブースト |
| `'晴れ'` | `'indoor'` | - | ×1.0 | 補正なし |
| `'曇り'` | - | - | ×1.0 | 補正なし |
| `'風強め'` | - | - | ×1.0 | 補正なし |

**注意事項:**
- 天気が `'雨'` or `'雪'` の場合、条件判定は上から順に評価される（`outdoor` → `indoor` → `weather_ok` の順）
- `weather_ok` は `includes('雨OK')` で部分一致判定
- `indoor_type` が `'both'` で `weather_ok` に `'雨OK'` が含まれない場合は補正なし（×1.0）

### 6.5 滞在時間妥当性補正

合計利用時間に対して1スポットあたりの適正滞在時間を算出し、DBの推定滞在時間が長すぎるスポットにペナルティを課す。

```
estimatedSpots = max(2, round(availableTime / 35))
perSpotTime = availableTime / estimatedSpots
```

| 条件 | 補正係数 | 説明 |
|------|---------|------|
| `stayMin > perSpotTime × 2` | **×0.5** | 大幅に長すぎる |
| `stayMin > perSpotTime × 1.5` (かつ上記に非該当) | **×0.8** | やや長い |
| `stayMin <= perSpotTime × 1.5` | ×1.0 | 適正 |

**具体例:**

| availableTime | estimatedSpots | perSpotTime | stayMin=60 | stayMin=45 | stayMin=30 |
|---|---|---|---|---|---|
| 120分 | max(2, round(120/35)) = max(2, 3) = 3 | 40分 | >80→×0.5 | >60→×0.8 | ≤60→×1.0 |
| 180分 | max(2, round(180/35)) = max(2, 5) = 5 | 36分 | >72→×0.5 | >54→×0.8 | ≤54→×1.0 |
| 60分 | max(2, round(60/35)) = max(2, 2) = 2 | 30分 | >60→×0.5 | >45→×0.8 | ≤45→×1.0 |

## 7. 多様性選択アルゴリズム

### 7.1 概要

`selectDiverseSpots` は、スコアリング済み候補プールから、カテゴリの多様性と地理的近接性を考慮しつつ、重み付きランダムでスポットを選択する逐次貪欲法アルゴリズムである。

### 7.2 選択数の算出

```
targetSpots    = clamp(round(availableTime / 35), 2, 6)
newSpotsNeeded = max(1, targetSpots - pinned.length)
poolCount      = min(newSpotsNeeded + 2, 8)
```

| availableTime | targetSpots | pinned=0の場合のnewSpotsNeeded | poolCount |
|---|---|---|---|
| 60分 | max(2, min(6, round(60/35))) = max(2, min(6, 2)) = 2 | 2 | min(4, 8) = 4 |
| 90分 | max(2, min(6, round(90/35))) = max(2, min(6, 3)) = 3 | 3 | min(5, 8) = 5 |
| 120分 | max(2, min(6, round(120/35))) = max(2, min(6, 3)) = 3 | 3 | min(5, 8) = 5 |
| 180分 | max(2, min(6, round(180/35))) = max(2, min(6, 5)) = 5 | 5 | min(7, 8) = 7 |
| 210分 | max(2, min(6, round(210/35))) = max(2, min(6, 6)) = 6 | 6 | min(8, 8) = 8 |
| 250分 | max(2, min(6, round(250/35))) = max(2, min(6, 7)) = 6 | 6 | min(8, 8) = 8 |

**poolCountがnewSpotsNeededより多い理由:** 多めに選択しておき、タイムテーブル生成時にdeadlineで打ち切ることで、時間に余裕がある場合により多くのスポットを含められるようにするため。

### 7.3 アルゴリズム詳細

#### ステップ1: 1つ目の選択

```
first = candidates[Math.floor(Math.random() × candidates.length)]
```

- 全候補から**均等ランダム**で1つ選択
- スコアは考慮しない（バウンディングボックスで既にエリアが絞られているため）
- 選択後、`usedGroups` に1つ目のグループを追加、`usedIds` に1つ目のIDを追加

#### ステップ2: 2つ目以降の選択

各ステップで以下の重みを計算し、重み付きランダムで選択する。

```
for each candidate c (usedIdsに含まれないもの):
  d = distanceKm(last.lat, last.lon, c.lat, c.lon)
  proximityScore = max(0.1, exp(-d × 0.7))
  diversityBonus = usedGroups.has(c.group) ? 0.05 : 1.0
  combo = sqrt(c.score) × proximityScore × diversityBonus
```

**各係数の特性:**

| 係数 | 数式 | 特性 | 目的 |
|------|------|------|------|
| `sqrt(score)` | `Math.sqrt(c.score)` | スコアを平坦化（ランダム性を上げる） | 高スコアスポットが独占しないようにする |
| `proximityScore` | `max(0.1, exp(-d × 0.7))` | 距離に応じた指数減衰、下限0.1 | 近いスポットを優先しつつ遠方も候補に残す |
| `diversityBonus` | 同カテゴリ: 0.05 / 異カテゴリ: 1.0 | 20倍の差 | カテゴリの多様性を強く優先 |

#### proximityScore の距離減衰テーブル

| 距離(km) | exp(-d × 0.7) | proximityScore (max(0.1, ...)) | 説明 |
|---|---|---|---|
| 0.0 | 1.000 | **1.000** | 同一地点 |
| 0.2 | 0.869 | **0.869** | 約2.5分(徒歩) |
| 0.5 | 0.705 | **0.705** | 約6分(徒歩) |
| 1.0 | 0.497 | **0.497** | 約12.5分(徒歩) |
| 1.5 | 0.350 | **0.350** | 約19分(徒歩) |
| 2.0 | 0.247 | **0.247** | 約25分(徒歩) |
| 3.0 | 0.122 | **0.122** | 約37.5分(徒歩) |
| 4.0 | 0.061 | **0.100** | 下限に到達 |

### 7.4 重み付きランダム選択

`weightedRandomPickFromPool` 関数は、重み配列に基づくルーレット選択を実行する。

```
total = sum(weights)
r = Math.random() × total
for i in 0..pool.length-1:
  r -= weights[i]
  if r <= 0: return pool[i]
return pool[pool.length - 1]  // フォールバック
```

- 重みが大きいスポットほど選択確率が高い
- 確定的ではなく、毎回異なるプランが生成される

### 7.5 終了条件

| 条件 | 動作 |
|------|------|
| `selected.length >= count` | ループ終了 |
| `pool.length === 0`（未使用候補が枯渇） | ループ終了 |
| `pick` が null | ループ終了 |

## 8. ルート最適化アルゴリズム

### 8.1 概要

`optimizeRoute` は、選択されたスポット群を最短距離（方向転換ペナルティ付き）で巡回する順序に並び替える。同一ビル内のスポットは1つのノードとしてグルーピングし、連続配置する。

### 8.2 同一ビルグルーピング

#### 8.2.1 ビル名パース（parseBuildingName）

住所文字列からビル・施設名を抽出する。

**正規表現:**

| 正規表現 | 説明 | 用途 |
|---|---|---|
| `/\d[-−]\d+\s+(.+)$/` | 番地(`数字-数字`)の後のスペース以降をビル名として抽出 | ビル名の抽出 |
| `/\s*[B]?\d+[F階].*$/i` | 階数表記（`3F`, `B1F`, `2階`等）とその後続を除去 | フロア部分の削除 |
| `/丁目付近$/` | 「丁目付近」で終わる住所をエリア名として判定 | ビル名ではないケースの除外 |

**パース手順:**

```
1. address が null → undefined 返却
2. 「丁目付近」で終わる → undefined 返却
3. BUILDING_RE でマッチ → マッチ部分からフロア情報を除去
4. マッチしない → undefined 返却
5. ビル名が空文字列 or 2文字未満 → undefined 返却
6. ビル名を返却
```

**パース例:**

| 住所 | 抽出結果 | 説明 |
|------|---------|------|
| `"東京都渋谷区神宮前1-2-3 表参道ヒルズ3F"` | `"表参道ヒルズ"` | フロア除去 |
| `"東京都渋谷区神宮前1-2-3 表参道ヒルズ"` | `"表参道ヒルズ"` | フロアなし |
| `"東京都渋谷区神宮前1-2-3"` | `undefined` | ビル部分なし |
| `"東京都渋谷区神宮前1丁目付近"` | `undefined` | エリア名 |

#### 8.2.2 グルーピングロジック

```
1. 全スポットを走査
2. building が一致するスポットのindexをグループ化
3. 同一ビルに2スポット以上 → 1つの RouteNode にまとめる（代表座標は最初のスポットの座標）
4. 同一ビルに1スポットのみ → ソロ扱い（通常のノード）
5. building が undefined → ソロ扱い
```

### 8.3 TSP（全順列探索）

#### 8.3.1 アルゴリズム

ノード数が少ない（最大8ノード程度）ため、全順列（N!）を列挙して最短距離の順列を選択する。

```
for each permutation perm of nodeIndices:
  totalDist = distanceKm(startLat, startLon, nodes[perm[0]].lat, nodes[perm[0]].lon)
  for i = 1 to perm.length - 1:
    totalDist += distanceKm(nodes[perm[i-1]], nodes[perm[i]])

  // 方向転換ペナルティを適用
  totalDist = applyTurnPenalty(totalDist, perm)

  if totalDist < bestDist:
    bestDist = totalDist
    bestOrder = perm
```

**計算量:** O(N!) where N = ノード数（最大8、8! = 40,320通り）

**出発地点:**
- `origin` が指定されている場合: `origin.lat`, `origin.lng` を出発地とする
- `origin` が未指定の場合: 最初のノード (`nodes[0]`) の座標を出発地とする

#### 8.3.2 方向転換ペナルティ

3ノード以上の順列に対して、連続する3ノード間の方向変化を検出し、急角度の方向転換にペナルティを課す。

```
for i = 1 to perm.length - 2:
  prevNode = (i === 1) ? origin : nodes[perm[i-2]]
  curNode  = nodes[perm[i-1]]
  nextNode = nodes[perm[i]]

  angle1 = atan2(curNode.lat - prevNode.lat, curNode.lon - prevNode.lon)
  angle2 = atan2(nextNode.lat - curNode.lat, nextNode.lon - curNode.lon)

  angleDiff = abs(angle2 - angle1)
  if angleDiff > π: angleDiff = 2π - angleDiff

  if angleDiff > π/2:  // 90度超
    totalDist *= 1 + (angleDiff / π) × 0.3
```

| angleDiff (rad) | angleDiff (deg) | ペナルティ倍率 | 説明 |
|---|---|---|---|
| π/2 (1.571) | 90 | 1.0（適用なし） | 直角丁度は閾値ちょうどで適用なし |
| π/2 + ε | 90+ | ≈ ×1.15 | 最小ペナルティ |
| 2π/3 (2.094) | 120 | × (1 + 0.667 × 0.3) = **×1.200** | 中程度の方向転換 |
| 3π/4 (2.356) | 135 | × (1 + 0.75 × 0.3) = **×1.225** | やや大きな方向転換 |
| π (3.142) | 180 | × (1 + 1.0 × 0.3) = **×1.300** | 完全なUターン |

**注意:** ペナルティは **totalDist に対する乗算** であるため、複数回の方向転換がある場合は累積される（乗算の乗算）。

### 8.4 同カテゴリ連続解消

ルート最適化後に、隣接する同一カテゴリのスポットを検出し、後方の異カテゴリスポットとスワップする。

```
for i = 0 to ordered.length - 2:
  if ordered[i].group === ordered[i+1].group:
    for j = i+2 to ordered.length - 1:
      if ordered[j].group !== ordered[i].group:
        swap(ordered[i+1], ordered[j])
        break  // 1回スワップしたら次のiへ
```

**制約:**
- 前方（i以前）のスポットは移動しない
- スワップ対象は `i+2` 以降から順に探索し、最初に見つかった異カテゴリとスワップ
- 異カテゴリが見つからない場合（全て同カテゴリ）、スワップは行われない
- このスワップによりルート距離が増加する場合がある（距離よりカテゴリ多様性を優先）

## 9. タイムテーブル生成

### 9.1 開始時刻の算出

```
startMinute = ceil(currentMinute / 15) × 15
```

| currentMinute | startMinute | 説明 |
|---|---|---|
| 0 | 0 | ちょうど |
| 1 - 15 | 15 | 15分単位に切り上げ |
| 16 - 30 | 30 | 15分単位に切り上げ |
| 31 - 45 | 45 | 15分単位に切り上げ |
| 46 - 59 | 60（= 次の時の 00分） | 15分単位に切り上げ |

### 9.2 タイムライン構築

```
baseDate = new Date()
baseDate.setHours(currentHour, startMinute, 0, 0)
startTime = baseDate.getTime()  // ミリ秒
currentTime = startTime
deadline = startTime + availableTime × 1.1 × 60 × 1000  // 10%の余裕
```

### 9.3 各スポットの処理

ルート最適化済みの順序で、各スポットについて以下を処理する。

```
for i = 0 to routed.length - 1:
  item = routed[i]
  isPinned = pinnedIds.has(item.spot.source_id)
  stayMin = item.spot.estimated_stay_min ?? 30

  // 移動時間算出
  walkMin = calculateWalkMin(i, item, routed, params)

  // 打ち切り判定（ピン留めは免除）
  if !isPinned:
    endTimeCheck = currentTime + walkMin × 60 × 1000 + stayMin × 60 × 1000
    if result.length >= 2 AND endTimeCheck > deadline:
      break

  // 時刻更新
  currentTime += walkMin × 60 × 1000
  spotTime = new Date(currentTime)  // 到着時刻
  currentTime += stayMin × 60 × 1000  // 滞在後の時刻

  // PlanSpot を result に追加
```

### 9.4 移動時間算出ルール

| 条件 | 移動時間（分） | 説明 |
|------|-------------|------|
| 同一ビル内（`item.building === routed[i-1].building`） | **0分** | ビル内移動は無視 |
| 最初のスポット + origin指定あり | `max(3, ceil(距離m / 80))` | 出発地からの徒歩 |
| 2つ目以降（同一ビルでない） | `max(3, ceil(距離m / 80))` | 前スポットからの徒歩 |
| 最初のスポット + origin未指定 | **0分** | 移動時間なし |

**最低移動時間:** 3分（同一ビル以外は、たとえ距離が近くても最低3分を確保）

### 9.5 打ち切り条件

| 条件 | 判定 | 動作 |
|------|------|------|
| ピン留めスポット | 打ち切り対象外 | **必ず含める** |
| 新規スポット + 結果2件未満 | 打ち切り対象外 | **最低2件は保証** |
| 新規スポット + 結果2件以上 + endTimeCheck > deadline | **打ち切り** | これ以降のスポットは追加しない |

**deadline の計算:**
```
deadline = startTime + availableTime × 1.1 × 60 × 1000
```
- `availableTime` の10%超過まで許容する
- 例: `availableTime = 120分` の場合、`deadline = startTime + 132分`

各スポットは以下の `PlanSpot` として返却される。

## 10. 営業時間判定

### 10.1 isOpen 関数仕様

```typescript
function isOpen(spot: DbSpot, hour: number, minute: number): boolean
```

### 10.2 判定フロー

```
[1] is_open_24h === true → return true
[2] starttime が未設定(null/undefined) → return true
[3] closetime が未設定(null/undefined) → return true
[4] starttime/closetime をパース → 分単位に変換
    cur  = hour × 60 + minute
    open = sH × 60 + sM
    close = cH × 60 + cM
[5] close < open（日付跨ぎ）の場合:
    return cur >= open || cur < close
[6] 通常の場合:
    return cur >= open && cur < close
[7] パースエラー時: return true（安全側に倒す）
```

### 10.3 決定表

| is_open_24h | starttime | closetime | 判定 |
|---|---|---|---|
| `true` | - | - | **営業中** |
| `false`/`null` | `null` | - | **営業中**（データ不足は安全側） |
| `false`/`null` | 設定あり | `null` | **営業中**（データ不足は安全側） |
| `false`/`null` | 設定あり | 設定あり | 時刻計算による判定 |

### 10.4 日付跨ぎのケース

**例: バー（starttime=18:00, closetime=02:00）**

```
open  = 18 × 60 + 0 = 1080
close = 2 × 60 + 0  = 120

close(120) < open(1080) → 日付跨ぎモード
  cur >= 1080 (18:00以降) → true
  cur < 120  (02:00未満) → true
  それ以外 → false
```

| 現在時刻 | cur(分) | 判定 |
|---|---|---|
| 01:00 | 60 | 60 < 120 → **営業中** |
| 03:00 | 180 | 180 >= 1080? No, 180 < 120? No → **営業時間外** |
| 12:00 | 720 | 720 >= 1080? No, 720 < 120? No → **営業時間外** |
| 18:00 | 1080 | 1080 >= 1080 → **営業中** |
| 23:00 | 1380 | 1380 >= 1080 → **営業中** |

### 10.5 通常ケース

**例: カフェ（starttime=09:00, closetime=21:00）**

```
open  = 9 × 60 + 0  = 540
close = 21 × 60 + 0 = 1260

close(1260) >= open(540) → 通常モード
  cur >= 540 AND cur < 1260 → true
```

| 現在時刻 | cur(分) | 判定 |
|---|---|---|
| 08:00 | 480 | 480 >= 540? No → **営業時間外** |
| 09:00 | 540 | 540 >= 540 AND 540 < 1260 → **営業中** |
| 15:00 | 900 | 900 >= 540 AND 900 < 1260 → **営業中** |
| 21:00 | 1260 | 1260 >= 540 AND 1260 < 1260? No → **営業時間外** |

## 12. 距離・時間計算式

### 12.1 Haversine公式（2点間距離）

```typescript
function distanceKm(lat1, lon1, lat2, lon2): number {
  R = 6371  // 地球の半径 (km)

  dLat = (lat2 - lat1) × π / 180
  dLon = (lon2 - lon1) × π / 180

  a = sin(dLat / 2)² + cos(lat1 × π / 180) × cos(lat2 × π / 180) × sin(dLon / 2)²

  distance = R × 2 × atan2(√a, √(1 - a))

  return distance  // 単位: km
}
```

### 12.2 定数

| 定数 | 値 | 説明 |
|------|-----|------|
| R | 6371 | 地球の平均半径（km） |
| 徒歩速度 | 80 m/分（4.8 km/h） | 日本の都市部における平均的な徒歩速度 |

### 12.3 徒歩時間換算

```
walkMinToKm(minutes) = minutes × 80 / 1000  [km]
walkMin = max(3, ceil(distanceKm × 1000 / 80))  [分]
```

| 距離(km) | 距離(m) | 徒歩時間(分) | max(3, ...) |
|---|---|---|---|
| 0.05 | 50 | ceil(50/80) = 1 | **3分** |
| 0.10 | 100 | ceil(100/80) = 2 | **3分** |
| 0.24 | 240 | ceil(240/80) = 3 | **3分** |
| 0.50 | 500 | ceil(500/80) = 7 | **7分** |
| 1.00 | 1000 | ceil(1000/80) = 13 | **13分** |
| 1.60 | 1600 | ceil(1600/80) = 20 | **20分** |

### 12.4 バウンディングボックス用換算

```
latDelta = walkKm / 111              [度]
lonDelta = walkKm / (111 × cos(lat × π / 180))  [度]
```

| 概数 | 値 | 説明 |
|------|-----|------|
| 緯度1度 | 約111 km | 地球の子午線周長 ÷ 360 |
| 経度1度 | 約111 × cos(lat) km | 緯度に依存 |

**東京（lat ≈ 35.68）での参考値:**

| walkRangeMinutes | walkKm | latDelta | lonDelta | 範囲(南北) | 範囲(東西) |

### D. 全順列生成関数

```typescript
function permutations(arr: number[]): number[][] {
  if (arr.length <= 1) return [arr];
  const result: number[][] = [];
  for (let i = 0; i < arr.length; i++) {
    const rest = [...arr.slice(0, i), ...arr.slice(i + 1)];
    for (const perm of permutations(rest)) {
      result.push([arr[i], ...perm]);
    }
  }
  return result;
}
```
