# NowGo 要件定義書

## 1. プロジェクト概要

### 1.1 サービス名
**NowGo**（ナウゴー）

### 1.2 コンセプト
「今すぐ出かけよう」をテーマに、ユーザーの現在地・空き時間・気分から最適なお出かけプランを自動生成するWebアプリケーション。新たなお店やスポットとの偶然の出会いを提供することを最大の価値とする。

### 1.3 ターゲットユーザー
- MVP段階では東京都内で気軽にお出かけしたい人
- 新しいお店やスポットを開拓したい人
- 予定を決めずにふらっと出かけたい人

### 1.4 提供価値
- 検索や計画の手間なく、すぐに出発できるプランが手に入る
- 自分では見つけられなかった隠れた名店・スポットとの出会い
- 歩いて回れる現実的なルート提案（移動時間・営業時間を考慮）

## 2. 機能要件

### 2.1 ユーザー認証機能（F-AUTH）

| ID | 機能 | 説明 |
|------|------|------|
| F-AUTH-01 | メール認証 | メールアドレスとパスワードによるサインアップ・ログイン |
| F-AUTH-02 | プロフィール自動作成 | サインアップ時にDBトリガーでユーザープロフィールを自動生成 |
| F-AUTH-03 | ゲストモード | アカウント不要で主要機能を利用可能（履歴・お気に入り保存は不可） |
| F-AUTH-04 | ログアウト | セッション破棄、認証画面へ遷移 |

### 2.2 プラン生成機能（F-PLAN）

| ID | 機能 | 説明 |
|------|------|------|
| F-PLAN-01 | 出発地設定 | GPS現在地取得 or 駅名検索（オートコンプリート）で出発地を指定 |
| F-PLAN-02 | 時間設定 | 遊べる時間を 60分〜1日 の範囲で選択 |
| F-PLAN-03 | 詳細条件設定 | モード・歩行範囲・日付・開始時刻・スタイル・天気・屋内外を選択 |
| F-PLAN-04 | プラン自動生成 | 条件に基づきDBからスポットを検索・スコアリング・ルート最適化してプランを生成 |
| F-PLAN-05 | プラン再作成 | 前回表示したスポットを除外して新しいプランを生成 |
| F-PLAN-06 | スポットピン留め | 気に入ったスポットをピン留めし、再作成時も維持 |
| F-PLAN-07 | 除外リストリセット | 候補が枯渇した場合、除外リストをリセットして再検索 |

### 2.3 プラン表示機能（F-VIEW）

| ID | 機能 | 説明 |
|------|------|------|
| F-VIEW-01 | プラン一覧表示 | スポットをカード形式で時系列順に表示 |
| F-VIEW-02 | 地図表示 | Leaflet地図上にスポットを番号付きマーカーで表示、ルートを点線で接続 |
| F-VIEW-03 | スポット詳細 | カテゴリ・到着時刻・滞在時間・説明文・Webサイトリンク・住所を表示 |
| F-VIEW-04 | 同一ビル表示 | 同じビル内のスポットは「同じ○○ビル内！」と表示 |
| F-VIEW-05 | 移動表示 | スポット間の移動を「てくてく歩いて...」等のテキストで表現 |

### 2.4 プラン実行機能（F-EXEC）

| ID | 機能 | 説明 |
|------|------|------|
| F-EXEC-01 | 現在スポット表示 | 実行中のスポットを大きく表示、進行状況をプログレスバーで表示 |
| F-EXEC-02 | Google Mapsナビ | 「スポットへ行く」ボタンでGoogle Maps（徒歩ナビ、現在地起点）を開く |
| F-EXEC-03 | 次のスポットへ | ワンタップで次のスポットに進む |
| F-EXEC-04 | プラン完了 | 全スポット完了時、訪問記録をDB保存 |
| F-EXEC-05 | 途中終了 | 確認ダイアログを経て途中でプランを終了 |

### 2.5 ユーザー管理機能（F-USER）

| ID | 機能 | 説明 |
|------|------|------|
| F-USER-01 | プロフィール編集 | ニックネーム・性別・生年月日を編集 |
| F-USER-02 | お気に入り管理 | お気に入りスポット数を表示 |
| F-USER-03 | 訪問履歴管理 | 行ったスポット数を表示 |
| F-USER-04 | プラン履歴管理 | 過去のプラン数を表示 |

### 2.6 データ管理機能（F-DATA）

| ID | 機能 | 説明 |
|------|------|------|
| F-DATA-01 | 直近駅記録 | 検索した駅を最大5件まで記録（次回表示用） |
| F-DATA-02 | 前回条件復元 | 前回の検索条件（駅・時間・人数）を保存・復元 |
| F-DATA-03 | 訪問記録 | プラン完了時に全スポットをvisited_spotsに記録 |
| F-DATA-04 | プラン履歴保存 | 完了したプランをplan_historyに保存 |

## 3. 非機能要件

### 3.1 パフォーマンス
- プラン生成: 3秒以内（DB検索500件 + スコアリング + ルート最適化）
- 画面遷移: 即座（SPA、クライアントサイドルーティング）

### 3.2 対応環境
- スマートフォンブラウザ（主要対象）
- デスクトップブラウザ（レスポンシブ対応）
- iOS Safari / Android Chrome

### 3.3 インフラ
- フロントエンド: Netlifyにデプロイ
- データベース: Supabase（PostgreSQL）
- 認証: Supabase Auth

### 3.4 セキュリティ
- Supabase RLS（Row Level Security）によるデータアクセス制御
- JWTベースのセッション管理

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| スポット | DB上の1つのお出かけ先（カフェ、ギャラリー、公園など） |
| プラン | 複数のスポットを時系列順に並べたお出かけ計画 |
| モード | プランの方向性（定番/新規開拓/冒険） |
| スコープ | スポットの規模（big=大型施設, medium=中規模, small=個別店舗） |
| famousLevel | スポットの知名度（1〜5、5が最も有名） |
| ピン留め | 再作成時も維持するスポットのブックマーク |
| バウンディングボックス | 出発地を中心とした矩形検索範囲 |
