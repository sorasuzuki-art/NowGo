# NowGo 状態遷移図

## 1. アプリケーション全体の状態遷移

Zustandの `currentScreen` で管理される画面状態の遷移。

```mermaid
stateDiagram-v2
    [*] --> Loading : アプリ起動
    Loading --> Auth : 未ログイン & 非ゲスト
    Loading --> Dashboard : ログイン済み or ゲスト

    state "認証" as Auth
    state "ダッシュボード" as Dashboard
    state "プラン確認" as Plan
    state "プラン実行" as Executing

    Auth --> Dashboard : signIn / signUp / signInAsGuest
    Auth --> Dashboard : ゲストに戻る（ゲストモード中のみ）

    Dashboard --> Plan : NowGo!（プラン生成成功）
    Dashboard --> Auth : ログアウト（ProfileScreen経由）

    Plan --> Executing : 「このプランで行く」
    Plan --> Plan : 「再作成」
    Plan --> Dashboard : 「戻る」（確認ダイアログ経由）

    Executing --> Executing : 「次のスポットへ」
    Executing --> Executing : 「前のスポットへ」
    Executing --> Dashboard : 「プラン完了」（全スポット訪問記録+履歴保存）
    Executing --> Dashboard : 「途中終了」（現在までの訪問記録のみ）
```

## 2. 認証状態の遷移

```mermaid
stateDiagram-v2
    [*] --> Checking : アプリ起動

    state "セッション確認中" as Checking
    state "未認証" as Unauthenticated
    state "ログイン済み" as Authenticated
    state "ゲスト" as Guest

    Checking --> Authenticated : セッションあり（profile取得成功）
    Checking --> Unauthenticated : セッションなし

    Unauthenticated --> Authenticated : signIn / signUp
    Unauthenticated --> Guest : signInAsGuest

    Authenticated --> Unauthenticated : signOut
    Guest --> Unauthenticated : 「会員登録・ログイン」
    Guest --> Guest : 「ゲストに戻る」（Auth画面→Dashboard）

    Guest --> Guest : アプリ操作（DB保存不可）
    Authenticated --> Authenticated : アプリ操作（DB保存可）
```

### 状態ごとの機能可否

| 機能 | 未認証 | ゲスト | ログイン済み |
|------|--------|--------|-------------|
| プラン生成 | - | OK | OK |
| プラン実行 | - | OK | OK |
| お気に入り保存 | - | - | OK |
| 訪問記録保存 | - | - | OK |
| プラン履歴保存 | - | - | OK |
| 直近の検索駅 | - | localStorage | DB |
| プロフィール編集 | - | - | OK |

## 3. プラン生成の状態遷移

```mermaid
stateDiagram-v2
    [*] --> Idle : DashboardScreen表示

    state "入力待ち" as Idle
    state "検索中" as Searching
    state "スコアリング" as Scoring
    state "候補選択" as Selecting
    state "ルート最適化" as Optimizing
    state "タイムテーブル生成" as Building
    state "プラン表示" as Displaying
    state "エラー" as Error

    Idle --> Searching : NowGo!ボタン押下
    Searching --> Scoring : DB検索完了（候補あり）
    Searching --> Searching : フォールバック検索（候補0件）
    Searching --> Error : フォールバックも0件
    Scoring --> Selecting : スコアリング完了
    Selecting --> Optimizing : スポット選択完了
    Optimizing --> Building : ルート最適化完了
    Building --> Displaying : タイムテーブル生成完了
    Error --> Idle : 再試行
```

## 4. プラン実行の状態遷移

```mermaid
stateDiagram-v2
    [*] --> Spot1 : 「このプランで行く」

    state "スポット1" as Spot1
    state "スポット2" as Spot2
    state "スポットN" as SpotN
    state "最終スポット" as Last
    state "完了確認" as ConfirmComplete
    state "途中終了確認" as ConfirmExit
    state "完了保存" as SavingComplete
    state "途中保存" as SavingAbort
    state "ダッシュボード" as Done

    Spot1 --> Spot2 : 「次のスポットへ」
    Spot2 --> SpotN : 「次のスポットへ」
    Spot2 --> Spot1 : 「前のスポットへ」
    SpotN --> Last : 「次のスポットへ」（最後の1つ前）
    SpotN --> Spot2 : 「前のスポットへ」
    Last --> SpotN : 「前のスポットへ」
    Last --> ConfirmComplete : 「プランを完了する」

    Spot1 --> ConfirmExit : 「終了する」/ ヘッダー戻る
    Spot2 --> ConfirmExit : ヘッダー戻る
    SpotN --> ConfirmExit : ヘッダー戻る
    Last --> ConfirmExit : ヘッダー戻る

    ConfirmComplete --> SavingComplete : 「完了する」
    ConfirmComplete --> Last : 「キャンセル」

    ConfirmExit --> SavingAbort : 「終了する」
    ConfirmExit --> Spot1 : 「キャンセル」（元のスポットに戻る）

    SavingComplete --> Done : 全スポット訪問記録 + プラン履歴 保存完了
    SavingAbort --> Done : 現在までのスポット訪問記録のみ保存
```

### プログレス表示

| 進捗 | ラベル |
|------|--------|
| 0% | スタート！ |
| 1〜33% | スタート！ |
| 34〜66% | 乗ってきた |
| 67〜99% | もうすぐ！ |
| 100% | コンプリート！ |

## 5. プラン再作成の状態遷移

```mermaid
stateDiagram-v2
    [*] --> PlanDisplayed : プラン表示中

    state "プラン表示中" as PlanDisplayed
    state "再生成中" as Regenerating
    state "除外リストリセット" as Resetting
    state "新プラン表示" as NewPlan

    PlanDisplayed --> Regenerating : 「再作成」ボタン
    Regenerating --> NewPlan : 候補あり → 新プラン生成
    Regenerating --> Resetting : 候補0件
    Resetting --> NewPlan : リセット後に候補あり
    Resetting --> PlanDisplayed : リセット後も候補0件（トーストでエラー表示）
    NewPlan --> PlanDisplayed : 表示完了
```

### 除外リストの状態

```
初回生成:
  shownSpotIds = [] → プラン生成 → shownSpotIds = [A, B, C]

1回目の再作成:
  excludeSpotIds = [A, B, C] → 新プラン生成 → shownSpotIds = [A, B, C, D, E, F]

2回目の再作成:
  excludeSpotIds = [A, B, C, D, E, F] → 新プラン生成 → shownSpotIds = [A, B, C, D, E, F, G, H, I]

N回目（候補枯渇時）:
  excludeSpotIds = [A, B, ..., X] → 候補0件
  → リセット: excludeSpotIds = [現在のプランのスポットのみ]
  → 再検索 → shownSpotIds = [現在のプランのスポットのみ]
```

## 6. ピン留め状態の遷移

```mermaid
stateDiagram-v2
    state "未ピン留め" as Unpinned
    state "ピン留め済み" as Pinned

    [*] --> Unpinned : スポット生成時

    Unpinned --> Pinned : ブックマークアイコンタップ
    Pinned --> Unpinned : ブックマークアイコンタップ

    Pinned --> Pinned : プラン再作成（維持される）
    Unpinned --> Unpinned : プラン再作成（新スポットに置き換え）
```

### ピン留め時の動作
1. Zustandの `currentPlan.pinnedSpots[]` にIDを追加/削除
2. ログイン時: `toggleFavoriteSpot()` でDB（favorite_spots）にも保存/削除
3. 再作成時: pinnedSpots はそのまま searchSpotsFromDB に渡され、必ずプランに含まれる

## 7. GPS取得の状態遷移

```mermaid
stateDiagram-v2
    [*] --> Idle

    state "未取得" as Idle
    state "取得中" as Fetching
    state "取得成功" as Success
    state "取得失敗" as Failed

    Idle --> Fetching : GPSボタンタップ
    Fetching --> Success : navigator.geolocation.getCurrentPosition 成功
    Fetching --> Failed : エラー（権限拒否/タイムアウト）

    Success --> Idle : リセット
    Failed --> Idle : リセット
    Failed --> Fetching : 再試行

    Success --> Success : 精度表示（±Xm）
```

### 取得成功時のデータ
```typescript
startLocation = {
  label: "現在地（±{accuracy}m）",
  lat: position.coords.latitude,
  lng: position.coords.longitude,
  source: 'gps',
  accuracy: position.coords.accuracy
}
```
